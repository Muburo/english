# レビュー仕様（正本）

> CLAUDE.md から切り出した詳細仕様（2026-07-02）。ライトレビュー・フルレビュー生成時にこのファイルを必ず読むこと。
> 分析の姿勢・例文ルールなどの共通ルールは CLAUDE.md 側にある。

## 定量指標（Progress Metrics）

レポートの冒頭に以下の指標を算出し、経時変化を追跡する。

| 指標 | 計算方法 | 見方 |
|------|---------|------|
| 言えなかった数 | 「言いたかったが言えなかった/間違えた表現」の総数 ÷ 対象エントリ数 | 少ないほど良い。表現力の向上を示す |
| 危険エラー数 | 意味が真逆・通じない表現の数（hardly, takes no side 等） | 0 が目標。最も重大な問題 |
| 導入バリエーション | 対象期間のスピーチ導入パターンの種類数 ÷ 全スピーチ数 | 1.0 に近いほど良い。同じ導入の繰り返しを検出 |
| 締めバリエーション | 対象期間のスピーチ締めパターンの種類数 ÷ 全スピーチ数 | 同上 |
| 時制エラー率 | 時制の誤り数 ÷ en_log の総文数（目視サンプリング） | 低いほど良い |
| 自覚率 | ja_log で自分で気づいたエラー数 ÷ 全エラー数 | 高いほど良い。自己分析力の指標 |
| 宣言実行率 | ja_log で宣言した目標のうち、後日の en_log で実行確認できた数 ÷ 宣言総数 | 高いほど良い。宣言と行動のギャップを可視化する（フルレビュー専用） |
| 構造完成率 | 意見スピーチで「立場→理由→具体例→再主張」の4要素中3つ以上を含むスピーチの割合 | 高いほど良い。意見スピーチの型の定着を測る（フルレビュー専用） |

#### 指標の運用
- **ライトレポート**: その期間の指標のみ表示
- **フルレポート**: 前回フルレポートとの比較を並べて表示（初回は比較なし）
- 指標は厳密な数値でなくてよい（目視カウント・概算OK）。傾向の把握が目的
- 1ヶ月、3ヶ月、半年、1年の節目で振り返りやすいよう、フルレポートに累積推移を記載する

## ライトレビュー（日次・週次）

**目的**: 復習しきれる量に絞り、即アクションに繋げる。

#### トリガー
- 「今日の分をレビューして」「今週の軽いレビュー」「ライトレビューして」
- 期間指定 + 「軽く」「簡単に」等の表現があればライトレビュー
- 1日〜1週間程度の期間が対象

#### 出力
`reports/quick-YYYY-MM-DD.md`（日次）または `reports/quick-YYYY-WXX.md`（週次）

**構成**:

**0. 指標スナップショット**
- 定量指標テーブル（その期間の値のみ、比較なし）
- 各指標に **100点満点のスコア** を付与し、上達のバロメーターにする

| 指標 | 100点（理想） | 0点（最低） | 配点の考え方 |
|------|-------------|-----------|------------|
| 言えなかった数 | 0個 | 10個以上 | 少ないほど高得点 |
| 危険エラー数 | 0個 | 3個以上 | 1個でも大きく減点（通じない英語は最重要） |
| 導入バリエーション | 毎回違うパターン | 毎回同じ | ユニーク率が高いほど高得点 |
| 締めバリエーション | 毎回違うパターン | 毎回同じ / 締めなし | 同上。締めなしは0点 |
| 時制エラー率 | 0% | 50%以上 | 低いほど高得点 |
| 自覚率 | 100% | 0% | 高いほど高得点 |

- 指標テーブルの各行に値とスコアを並記する（例: `4/18文 ≒ 22% → 65点`）
- **総合スコア** は全指標の加重平均で算出する（危険エラーの比重を最も高くする）
- 総合スコアに対応する **CEFR レベル** も併記する

| 点数 | CEFR | 目安 |
|------|------|------|
| 90-100 | B2+ | 複雑な話題でも正確に意見を述べられる |
| 75-89 | B2 | 多少のミスはあるが意見を論理的に展開できる |
| 60-74 | B1+ | 身近な話題なら意見を言えるが、表現の幅が狭い |
| 45-59 | B1 | 言いたいことの骨格は伝わるが、崩れや抜けが目立つ |
| 30-44 | A2+ | 簡単な意見は言えるが、意味が通じない箇所が複数ある |
| 0-29 | A2 | 単語の羅列に近く、文として成立していない箇所が多い |

**1. 今回のトップ3（言えなかった表現）**
- 最も重要な3つに絞る（汎用性・自覚度・危険度を総合判断）
- 各項目に以下を含める:
  - 言いたかったこと（日本語）
  - 実際の発話（en_log の引用。言えなかった場合は ja_log の記述）
  - **完成形の例文2つ**（フルセンテンスで。好みや言いやすさで選べるように）
  - 一行解説

**2. 今回のフォーカス弱点（1つだけ）**
- その期間で最も目立った弱点パターンを1つ選ぶ
- en_log の実際の発話 → **正しい完成形のフルセンテンス2つ**を対比で2〜3例
- 覚えるルール（1〜2行で簡潔に）

**3. 意見スピーチの弱点（1つだけ）**
- 意見を述べる・主張するタイプのスピーチに特有の弱点を1つ選ぶ
- 以下の3要素を必ず含める:
  1. **自分の根拠**: ja_log での本人の自覚や分析を引用し、なぜこの部分が弱いのかを示す
  2. **現状の発話**: en_log で実際にどう言ったかを引用し、何が足りないか・どう崩れているかを具体的に指摘する
  3. **理想の言い方**: 完成形のフルセンテンスを**2つ**提示（好みや場面で選べるように）

## フルレビュー（月次）

**目的**: 表現レベルの個別フィードバック（ライトレビューが担当）ではなく、大局的な構造課題・宣言の実行状況・根本原因を分析し、練習方法そのものを改善する。

#### トリガー
- 「今月のフルレビュー」「まとめて分析して」「フルレポート出して」
- 2週間以上の期間が対象の場合はデフォルトでフルレビュー
- 期間が指定されたら、その範囲内の日付フォルダのみを対象にする
- 期間指定がない場合は、前回フルレポート以降の未分析分を対象にする

#### 出力
`reports/review-YYYY-MM.md`（月次）または `reports/review-YYYY-WXX-WXX.md`（複数週）

#### ライトレビューとの役割分担

| 観点 | ライトレビュー | フルレビュー |
|------|-------------|------------|
| 粒度 | 表現レベル | 構造・戦略レベル |
| 表現リスト | トップ3（直近ベース） | トップ5（月横断・頻度×汎用性×反復ベース） |
| Anki素材 | 提供する | トップ5のみ提供 |
| 弱点分析 | その日/週の目立った弱点1つ | 月横断の根本原因（最大3つ） |
| アクション | 「この表現を覚える」 | 「練習方法をこう変える」+ トップ5を最優先暗記 |

**重要**: フルレビューにカテゴリ別の表現テーブルや Anki 用リストの大量列挙を含めない。それはライトレビューの役割。フルレビューは「なぜ言えないのか」の構造分析と「どう練習すれば変わるか」の戦略提示に徹する。

#### 構成

**0. 指標ダッシュボード**
- 全指標テーブル（既存6指標 + 宣言実行率 + 構造完成率）
- 今回の値 + 前回フルレポートの値を並べて比較（初回は比較なし）
- 変化の方向（↑↓→）を付記
- 100点満点スコア + CEFR レベル（ライトレビューと同じ基準）
- 特筆すべき変化があれば一言コメント

**1. 今月のトップ5（最優先で覚えるべき表現）**

ライトレビューのトップ3（直近で言えなかったもの）とは**選定基準が異なる**。月を通して浮き上がった最重要項目を、以下の3基準で厳選する:

| 選定基準 | 意味 | 表記 |
|---------|------|------|
| 汎用性 | これが使えたら多くの場面で効いただろう | (汎) |
| 反復宣言 | ja_log で繰り返し「言えるようにしたい」と振り返っている | (宣) + 出現回数・該当日 |
| 反復エラー | 複数エントリで同じミスを繰り返している | (反) + 出現回数・該当日 |

- 5つに絞る。3基準のうち**いずれかに該当**すれば候補。複数基準に該当するものを優先
- 各項目に以下を含める:
  - 順位と選定基準タグ（例: 1. (宣)(反) ）
  - 言いたかったこと + 実際の発話（en_log 引用）
  - **完成形の例文2つ**（Anki にそのまま入れられる完全な英文）
  - なぜトップ5に入ったか（選定理由を1〜2行で）

**2. 宣言 vs 実行（有言実行トラッカー）**

ja_log から「〜できるようにしたい」「〜を覚えておく」「〜を練習する」等の**目標宣言を全て抽出**し、その後の en_log で実行されたかを追跡する。

各宣言に以下のステータスを付与:

| ステータス | 意味 | 表示 |
|-----------|------|------|
| 達成 | 後日の en_log で正しく使えている | ✅ |
| 挑戦中 | 試みたが不完全、または一度だけ | ⚠️ |
| 未着手 | 宣言後、en_log に一度も出現していない | ❌ |
| 常連 | 3回以上の別の機会で同じ目標を宣言しているが未達成 | 🔁 |

- 宣言を時系列でリスト化（宣言日、内容、ステータス、実行確認日）
- **「常連」（🔁）をハイライト**し、なぜ定着しないかを分析
- 達成した宣言は成長の証拠として肯定的に取り上げる
- 宣言実行率を算出（ダッシュボードの指標と連動）

**3. 根本課題マップ（最大3つ）**

ライトレビューで個別に拾われたエラーや弱点を横断的に分析し、表面的な「言えなかった」の裏にある**根本原因**を特定する。**最大3つ**に絞る（多すぎると何も改善しない）。

各根本課題に以下を含める:
- **課題名**: 一言で（例:「認知負荷が上がると文法の自動化が崩壊する」）
- **表面に出る症状**: どういう形でスピーチに現れるか（複数の症状をまとめる）
- **根拠**: en_log の具体例を複数引用し、パターンの再現性を示す
- **なぜ起きるのか（原因仮説）**: 日本語の干渉、認知負荷、練習方法の問題、知識 vs 運用のギャップ等
- **自覚度**: ja_log で触れているか、完全に無意識か
- **前月からの変化**: 改善傾向か、停滞か、悪化か

課題の選び方:
- 個別の表現（「important を12回言った」）ではなく、その裏にある構造的問題（「形容詞に頼る思考パターン」「語彙の引き出しが1分野1個」）を特定する
- 複数のエントリで再現していること（1回限りのミスは対象外）
- 改善した場合のインパクトが大きいものを優先

**4. 今月の変化**

前回フルレポートと比較して、具体的に何が変わったかを分析する。

- **成長した点**: en_log から「前月にはなかった表現・構造・試み」を証拠付きで提示。小さな変化でも拾う
- **停滞・後退した点**: 前回のフルレビューで指摘した課題のうち、改善が見られないものを正直に指摘
- **新たに出てきた課題**: トピック難易度の上昇や新しい挑戦に伴って顕在化した課題

トーン: 成長を先に出し、停滞は「次の伸びしろ」として提示する。ただし正直であること。

**5. 来月の戦略（最大3つ）**

「この表現を覚えろ」ではなく「**練習の仕方をこう変えろ**」という戦略レベルのアドバイス。最大3つ。4つ以上出さない（3つ全てに取り組めなくても、1つ集中して改善する方が価値がある）。

各戦略に以下を含める:
- **戦略名**: 一言で（例:「導入テンプレート固定ドリル」）
- **対応する根本課題**: セクション3のどの課題に対応するか
- **なぜこの戦略か**: 現状のやり方では改善しない理由
- **具体的な練習方法**: 何を、いつ、どのように練習するか（抽象論禁止）
- **成功の判定基準**: 次回フルレビューで何が確認できれば「改善した」と言えるか

#### トーンの原則（ライトレビューとの違い）

- **ライトレビュー** = 表現コーチ（「この言い方を覚えよう」）
- **フルレビュー** = 戦略コンサルタント（「練習の仕方を変えよう」）
- 月を通じたパターンと構造に焦点を当てる
- ja_log の宣言と en_log のパターンの対比を重視する
- 厳しめ・踏み込んだ指摘歓迎だが、構造的な視点で
