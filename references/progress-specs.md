# 進化レポート仕様（月次プログレストラッキング）

> 目的: **変化の過程を客観的に追えるようにする**。進化していない月はしていないと正直に書く（本人合意済み）。
> 励ましではなく事実と証拠。読み返した本人が自分で気づくための材料を出すことがゴール。

## トリガーと頻度

- トリガー: 「英語の進化レポート」「進化レポート出して」「英語の月次進化」
- 頻度: 月1回。月初に前月分を生成するのが基本形
- 対象期間: 前月1ヶ月（初回・明示指定時は複数月も可）

## データソース

1. `reports/quick-YYYY-MM-*.md` — 当月と前月の全ライトレビュー
2. `YYYY-MM/*/en_log.md` / `ja_log.md` — 月初・月中・月末から3〜5日分を精読（引用の証拠採取）
3. `progress/TIMELINE.md` — 過去の推移（生成後に当月行を追記）
4. `history/PRACTICE-HISTORY.md` — 当月に練習方法が変わっていれば反映

## 定量指標の扱い（重要）

- ライトレビューの総合スコアは**レビューごとに採点基準・重み付けが揺れている**ため、月平均は参考値として扱う
- 生ログから機械的に出せる安定指標（練習日数・en_log 平均語数）を必ず併記して補正する
- 計算例: `wc -w` で語数、日付フォルダ数で練習日数

## レポート構成

```markdown
# 進化レポート — YYYY-MM

## 0. 月次スナップショット
練習日数 / 平均語数 / ライトレビュー平均スコア(参考値) / CEFR 帯（最頻値と範囲）/ 前月比の増減

## 1. 前月との差分（証拠つき）
伸びた点・停滞した点・新たに出た課題を各1〜3個。
必ず en_log の実発話を引用して証拠にする。証拠のない主張は書かない。

## 2. 弱点の生存分析
前月のフォーカス弱点が今月も出たか（再発 / 解消 / 変異）。
前月トップ3表現の「その後」— 後続スピーチで実際に使われた形跡があるか。

## 3. 変わらないもの（ベースライン）
数ヶ月動いていない指標・癖を1〜2個。停滞の正直な記録。

## 4. 見返し用の問い（2〜3個）
本人が読み返して自分で気づくためのオープンクエスチョン。アドバイスにしない。
```

- 長さ目安: 60〜100行。ライトレビューより俯瞰、フルレビュー（引退済み）より軽く
- 分析の姿勢は CLAUDE.md「分析の共通ルール」に準拠（厳しめ歓迎・抽象論禁止・引用ベース）

## 出力とルーチン（1回の実行で全部やる）

1. `progress/progress-YYYY-MM.md` に保存
2. `progress/TIMELINE.md` に当月の行を追記（一言コメント付き）
3. Obsidian `<vault>/100_Notes/1.YYYY年/進化レポート — YYYY-MM.md` に同一内容のノートを作成（`—` は em-dash）
4. Obsidian `<vault>/04_Life/5.💬英語/歩み/進化レポート.md`（インデックス）にリンクを追記し、TIMELINE 表を同期する
5. 練習方法が変わった月は `history/PRACTICE-HISTORY.md` と Obsidian 歩みノートの年表にも追記
6. **表現ストックを更新する**（下記「表現ストック更新」参照。レポート生成とワンセット）
7. git commit（`docs: YYYY-MM の進化レポートを作成`）+ push

## 表現ストック更新（月次レポートとワンセット／単体トリガー「表現ストック更新して」）

正本: `history/EXPRESSION-STOCK.md` ／ Obsidian ビュー: `04_Life/5.💬英語/歩み/表現ストック.md`（内容同一・両方更新）。

4カテゴリの昇格パイプライン: **4（言いたい）→ 3（今月言えた）→ 1（定着ストック）**。2（よく使う型）は頻度観察。

手順:
1. 現行ストックと当月の en_log / ja_log / quick-*.md を読む
2. **カテゴリ3（前月の新規）の定着判定**: 当月も複数日で正しく出現 → カテゴリ1へ昇格。出現なし → カテゴリ4へ戻す（状態=「一度出たが未定着」）
3. **カテゴリ3を当月の新規で書き換え**: 当月初めて正しい形で出現した表現（初出日付き）
4. **カテゴリ4の消し込みと補充**: 実際に出現したものは3へ昇格して削除。当月の ja_log 宣言・レビュートップ3から新規追加。退化（一度言えたのに崩れた）は状態欄を更新
5. **カテゴリ2の見直し**: 新しい持ちネタの追加、テンプレ固定化の懸念を備考に
6. 末尾の更新履歴に1行追記（日付＋昇格・追加・削除の要約）
7. Obsidian 側ビューに同一内容を反映

ルール: 証拠（日付・実発話）のない行は書かない。水増ししない（増えない月はそれが事実）。カテゴリ1は原則減らさない（退化が確認されたら状態注記付きで4へ降格してよい）。
