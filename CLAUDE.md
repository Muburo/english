# english — 英語スピーチ練習ログ

GitHub: `Muburo/english`（public）

## プロジェクト概要

英語スピーチ練習アプリで毎日 2 分程度即興で話した後の振り返りログ。
日本語で「言えなかったこと」「詰まった理由」「覚えておきたい表現」を音声日記として記録している。
文章は未整理で、思考途中・言い直し・迷いを含む。

## データ構造

```
english/
├── CLAUDE.md
├── .gitignore
├── plans/          # Claude Code 作業ファイル（一時的）
└── YYYY-MM-DD/     # 日付フォルダ（1日1フォルダ）
    ├── en_log.md   # 英語スピーチの書き起こし
    └── ja_log.md   # 日本語での振り返り（音声入力）
```

### ファイル命名規則
| ファイル | 命名 | 内容 |
|---------|------|------|
| 日付フォルダ | `YYYY-MM-DD` | 練習日ごとに1フォルダ |
| 英語ログ | `en_log.md` | 英語スピーチの書き起こし（音声認識） |
| 日本語ログ | `ja_log.md` | 日本語での振り返り（音声入力） |

### データの特徴
- `en_log.md`: 即興スピーチのため文法・構成が不完全。音声認識の誤りも含む
- `ja_log.md`: 口語的な音声書き起こし。句読点が不完全、思考の飛躍あり
- 一部の日付フォルダには `en_log.md` のみ（`ja_log.md` なし）の場合あり

## スピーキングのコンテキスト

- 一人練習（聴衆なし）
- 時間は約 2 分と短く、瞬時に構成を組み立てる必要がある
- 正確さよりも流暢さ・自然さを重視している

## 学習の目標

- 英語を「考えながらでも止まらずに話せる」ようになること
- 完璧な表現より、軽く・自然に言い切ることを優先
- 日本語では言えている考えを、英語でも即座に出せるようにしたい

## 分析機能

### 共通ルール

#### 分析の姿勢
- en_log（英語書き起こし）と ja_log（日本語振り返り）の**両方**を必ず読む
- ja_log で本人が「言えなかった」「言えるようにしたい」と自覚しているポイントを優先的に拾う
- 音声認識ミス（protein nail → protein shake 等）と本当の語彙不足を区別する
- 一般的な英語学習アドバイスは不要。このログ特有の「言い切れなさ」「詰まり方」「逃げ」を重視する
- なぜこの表現が出てこなかったのかを推測してよい
- 抽象論ではなく、再利用可能な形に分解する
- 厳しめ・踏み込んだ指摘歓迎。「できていない前提」で分析する

#### 全セクション共通
- 全てのセクションで「実際に言えなかった表現・間違えた表現」を起点にする
- 各項目に**完成形の例文**を必ず含める。例文は**フルセンテンス**（そのまま暗記して使える完全な英文）で書く。フレーズだけ・単語だけで終わらせない
- 完成形の例文は**2つ**出す（好みや言いやすさで選べるようにする）
- en_log の実際の発話と ja_log の自覚ポイントの両方を根拠にする

### 定量指標（Progress Metrics）

レポートの冒頭に以下の指標を算出し、経時変化を追跡する。

| 指標 | 計算方法 | 見方 |
|------|---------|------|
| 言えなかった数 | 「言いたかったが言えなかった/間違えた表現」の総数 ÷ 対象エントリ数 | 少ないほど良い。表現力の向上を示す |
| 危険エラー数 | 意味が真逆・通じない表現の数（hardly, takes no side 等） | 0 が目標。最も重大な問題 |
| 導入バリエーション | 対象期間のスピーチ導入パターンの種類数 ÷ 全スピーチ数 | 1.0 に近いほど良い。同じ導入の繰り返しを検出 |
| 締めバリエーション | 対象期間のスピーチ締めパターンの種類数 ÷ 全スピーチ数 | 同上 |
| 時制エラー率 | 時制の誤り数 ÷ en_log の総文数（目視サンプリング） | 低いほど良い |
| 自覚率 | ja_log で自分で気づいたエラー数 ÷ 全エラー数 | 高いほど良い。自己分析力の指標 |

#### 指標の運用
- **ライトレポート**: その期間の指標のみ表示
- **フルレポート**: 前回フルレポートとの比較を並べて表示（初回は比較なし）
- 指標は厳密な数値でなくてよい（目視カウント・概算OK）。傾向の把握が目的
- 1ヶ月、3ヶ月、半年、1年の節目で振り返りやすいよう、フルレポートに累積推移を記載する

### ライトレビュー（日次・週次）

**目的**: 復習しきれる量に絞り、即アクションに繋げる。

#### トリガー
- 「今日の分をレビューして」「今週の軽いレビュー」「ライトレビューして」
- 期間指定 + 「軽く」「簡単に」等の表現があればライトレビュー
- 1日〜1週間程度の期間が対象

#### 出力
`reports/quick-YYYY-MM-DD.md`（日次）または `reports/quick-YYYY-WXX.md`（週次）

**構成**:

**0. 指標スナップショット**
- 定量指標テーブル（その期間の値のみ、比較なし）
- 各指標に **100点満点のスコア** を付与し、上達のバロメーターにする

| 指標 | 100点（理想） | 0点（最低） | 配点の考え方 |
|------|-------------|-----------|------------|
| 言えなかった数 | 0個 | 10個以上 | 少ないほど高得点 |
| 危険エラー数 | 0個 | 3個以上 | 1個でも大きく減点（通じない英語は最重要） |
| 導入バリエーション | 毎回違うパターン | 毎回同じ | ユニーク率が高いほど高得点 |
| 締めバリエーション | 毎回違うパターン | 毎回同じ / 締めなし | 同上。締めなしは0点 |
| 時制エラー率 | 0% | 50%以上 | 低いほど高得点 |
| 自覚率 | 100% | 0% | 高いほど高得点 |

- 指標テーブルの各行に値とスコアを並記する（例: `4/18文 ≒ 22% → 65点`）
- **総合スコア** は全指標の加重平均で算出する（危険エラーの比重を最も高くする）
- 総合スコアに対応する **CEFR レベル** も併記する

| 点数 | CEFR | 目安 |
|------|------|------|
| 90-100 | B2+ | 複雑な話題でも正確に意見を述べられる |
| 75-89 | B2 | 多少のミスはあるが意見を論理的に展開できる |
| 60-74 | B1+ | 身近な話題なら意見を言えるが、表現の幅が狭い |
| 45-59 | B1 | 言いたいことの骨格は伝わるが、崩れや抜けが目立つ |
| 30-44 | A2+ | 簡単な意見は言えるが、意味が通じない箇所が複数ある |
| 0-29 | A2 | 単語の羅列に近く、文として成立していない箇所が多い |

**1. 今回のトップ3（言えなかった表現）**
- 最も重要な3つに絞る（汎用性・自覚度・危険度を総合判断）
- 各項目に以下を含める:
  - 言いたかったこと（日本語）
  - 実際の発話（en_log の引用。言えなかった場合は ja_log の記述）
  - **完成形の例文2つ**（フルセンテンスで。好みや言いやすさで選べるように）
  - 一行解説

**2. 今回のフォーカス弱点（1つだけ）**
- その期間で最も目立った弱点パターンを1つ選ぶ
- en_log の実際の発話 → **正しい完成形のフルセンテンス2つ**を対比で2〜3例
- 覚えるルール（1〜2行で簡潔に）

**3. 意見スピーチの弱点（1つだけ）**
- 意見を述べる・主張するタイプのスピーチに特有の弱点を1つ選ぶ
- 以下の3要素を必ず含める:
  1. **自分の根拠**: ja_log での本人の自覚や分析を引用し、なぜこの部分が弱いのかを示す
  2. **現状の発話**: en_log で実際にどう言ったかを引用し、何が足りないか・どう崩れているかを具体的に指摘する
  3. **理想の言い方**: 完成形のフルセンテンスを**2つ**提示（好みや場面で選べるように）

### フルレビュー（月次）

**目的**: 対象期間の全体像を把握し、弱点パターンと成長を可視化する。

#### トリガー
- 「今月のフルレビュー」「まとめて分析して」「フルレポート出して」
- 2週間以上の期間が対象の場合はデフォルトでフルレビュー
- 期間が指定されたら、その範囲内の日付フォルダのみを対象にする
- 期間指定がない場合は、前回フルレポート以降の未分析分を対象にする

#### 出力
`reports/review-YYYY-MM.md`（月次）または `reports/review-YYYY-WXX-WXX.md`（複数週）

**構成**:

**0. 指標ダッシュボード**
- 定量指標テーブル（今回の値 + 前回フルレポートの値を並べて比較）
- 初回は比較なし。2回目以降は変化の方向（↑↓→）を付記
- 特筆すべき変化があれば一言コメント

**1. 汎用性トップ10（最優先で覚えるべき表現）**
- 対象期間内で「言えなかった・間違えた表現」から、**他の場面でも使い回せる汎用性が高い表現**をランキング形式で10個選出
- 各項目に以下を含める:
  - 順位
  - 実際に言えなかった/間違えた場面（日本語で何を言いたかったか + en_log での実際の発話）
  - **完成形の例文**（そのまま Anki に入れられる完全な英文）
  - どんな場面で使えるか
  - なぜ汎用性が高いか
  - 出典（日付 + en/ja）

**2. 覚えるべき表現（Anki用・完成形つき）**
- カテゴリ別テーブル（日常系、意見系、仕事系、感情系、旅行系、人生系 等）
- 各項目に以下を含める:
  - 言いたかったこと（日本語）
  - 実際の発話（en_log からの引用。言えなかった場合は ja_log の記述）
  - **完成形の例文**（そのまま Anki に入れられる完全な英文）
  - 解説（なぜ出てこなかったか、正しい表現のポイント）
  - 出典（日付 + en/ja）

**3. できていない根拠（弱点パターン）**
- 各パターンに**自覚度**（ja_log で触れているか）を明記
- 具体例: en_log の**実際の発話を引用** → **正しい完成形の例文**を対比
- 弱点の原因分析と覚えるべきルール・代替表現（全て完成形の例文つき）

**4. 次の期間で重点的に回すべきもの**
- 優先度つきアクションリスト
- 各項目に**練習用の完成形例文**を含める（抽象的な指示で終わらせない）
